在标准盒模型中，如果给盒子设置width和height，实际设置的是 content box。 padding 和 border再加上设置的宽高一起决定了整个盒子的大小。 在怪异盒模型中，所有宽度都是可见宽度，所以内容宽度是该宽度减去边框和填充部分 变成怪异盒模型的方法 box-sizeing: border-box
box-sizeing属性
content-box: width height -> content 宽高只是来决定content的大小
border-box: width height -> content + padding + border
变成怪异盒模型的应用：
应用1： 量取尺寸时不用去计算一些值
应用2： 解决一些需要设置百分比的情况和盒模型值
所以，border-box在移动端应用很常见
